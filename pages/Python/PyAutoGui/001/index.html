01.Windows11下PyAutoGui的安装和使用

# 安装PyAutoGUI库

RPA(Robotic Process Automation)，即机器人流程自动化。 RPA技术用于将工作中可重复的部分流程化，让机器替我们完成这一工作。 PyAutoGUl使我们可以使用Python脚本来控制鼠标和键盘来自动与其他应用程序交互，我们可以用这个库来记录我们的键鼠操作，使得计算机像人一样去使用相应应用程序，将人从大量的重复性工作中解放出来。 以下将在自己家用的Window11家用中文版笔记本上，安装并试用PuAutoGUI库。

# 系统环境

操作系统：Windows11 家庭中文版 Python：3.11.2

# 使用Pip安装PyAutoGUI

先创建虚拟环境：

 virtualenv -p python3.11 PyRPA

而后进入虚拟环境目录下的Scripts文件夹，激活虚拟环境

cd PyRPA
cd Scripts
.\activate

备注：家用版Windows在激活虚拟环境 .\activate 时可能出现【在此系统中禁止执行脚本】的错误。这是由于系统现有执行策略可能是Restricted(默认设置)的原因。Restricted 策略不允许任何脚本运行。 要了解现有系统执行策略，可执行命令：

get-executionpolicy

要在本地计算机上运行您编写的未签名脚本和来自其他用户的签名脚本,可执行命令:

set-executionpolicy remotesigned

将执行策略更改为 remotesigned

激活虚拟环境后，在环境中安装PyAutoGui：

pip install pyautogui

查看是否已安装成功:

# 虚拟环境中引入pyAutoGUI

python
import pyautogui

# PyAutoGui的使用实例

# 常规函数

pyautogui.position()  # 当前鼠标所在的x、y坐标
pyautogui.size()  # 当前屏幕宽高
pyautogui.onScreen(x, y)  # 如果x\y指定的点落在屏幕内则返回True.使用时x、y需要赋值

# 故障保护(Fail-Safes)

在每次PyAutoGUI调用后设置2.5秒的暂停：

pyautogui.PAUSE = 2.5

当故障保护(Fail-Safes)模式为True时，将鼠标移到左上角将引发pyautogui.FailSafeException，该异常可能会中止程序：

pyautogui.FAILSAFE = True

# 鼠标控制

XY坐标的原点为0，0，位于屏幕的左上角。X向右增加，Y向下增加。

pyautogui.moveTo(x, y, duration=num_seconds)  # 鼠标在num_second内移动至 X、Y 所指定的坐标点
pyautogui.moveRel(xOffset, yOffset, duration=num_seconds)  # 鼠标在num_second内从当前坐标点依据 xOffset, yOffset 所指定的跨度进行相对移动

如果持续时间 duration 参数为0或未指定，则移动是立即的。

注意：在Mac上拖动不能立即进行。

pyautogui.dragTo(x,y,duration=num_seconds)#将鼠标拖动到XY
pyautogui.dragRel(xOffset,yOffset,duration=num_seconds)#相对于鼠标当前位置拖动鼠标

调用click()只需在鼠标当前位置用左键单击鼠标一次，但关键字参数可以改变这一点：

pyautogui.click(x=moveToX,y=moveToY,clicks=num_of_clicks,interval=secs_between_clicks,button='left')

button关键字参数可以是“left”、“middle”或“right”。

所有的点击都可以用click()完成，但这些函数的存在是为了可读性。关键字参数是可选的：

pyautogui.rightClick(x=moveTox,y=moveToy)
pyautogui.middleClick(x=moveTox,y=moveToy)
pyautogui.doubleClick(x=moveTox,y=moveToy)
pyautogui.tripleClick(x=moveTox,y=moveToy)

amount_to_scroll参数为正数将向上滚动，为负数将向下滚动：

pyautogui.scroll(amount_to_scroll,x=moveTox,y=moveToy)

鼠标按键向下和向上事件可以分别调用：

pyautogui.mouseDown(x=moveTox,y=moveToy,button='left')
pyautogui.mouseUp(x=moveTox,y=moveToy,button='left')

# 键盘控制

在函数调用时，按键指向键盘光标所在的位置。

pyautogui.typewrite('Hello world!\n', interval=secs_between_keys) #用于输入文本，换行符为Enter

还可以传递按键名称列表：

pyautogui.typetype([a','b','c','left','backspace','enter','f1'],interval=secs_between_keys)

键盘按键名称的完整列表位于pyautogui.KEYBOARD_KEYS中。

像Ctrl-S或Ctrl-Shift-1这样的键盘热键可以通过将键名列表传递给热键()来完成：

pyautogui.hotkey('ctrl', 'c')  # ctrl-c复制
pyautogui.hotkey('ctrl', 'v')  # ctrl-v粘贴

单个按键按下和放开事件可以分别调用：

pyautogui.keyDown(key_name)
pyautogui.keyUp(key_name)

# 消息框的控制

如果您需要暂停程序，直到用户单击“确定”，或者想向用户显示一些信息，那么消息框函数的名称与JavaScript类似：

pyautogui.alert('This displays some text with an OK button.')
pyautogui.confirm('This displays text and has an OK and Cancel button.')
pyautogui.prompt('This lets the user type in a string and press OK.')

如果用户单击“取消”，prompt（）函数将返回None。

# 屏幕截图功能

PyAutoGUI使用Pillow/PIL用于其图像相关数据处理。

在Linux上，您必须运行sudo apt-get-install scrum才能使用屏幕截图功能。

pyautogui.screenshot()  # 返回一个 Pillow/PIL 图像对象
pyautogui.screenshot('foo.png')  # 返回一个 Pillow/PIL 图像对象, 并保存到指定文件

如果你想要截图部分屏幕区域，你可以用locateOnScreen()方法在屏幕上获取。

pyautogui.locateOnScreen('looksLikeThis.png')  # 按照 (left, top, width, height) = (863, 417, 70, 13) 的格式获取屏幕区域图像

locateAllOnScreen（）函数将为其在屏幕上找到的所有窗体区域返回生成器：

>>> for i in pyautogui.locateAllOnScreen('looksLikeThis.png')
...
...
(863, 117, 70, 13)
(623, 137, 70, 13)
(853, 577, 70, 13)
(883, 617, 70, 13)
(973, 657, 70, 13)
(933, 877, 70, 13)

list(pyautogui.locateAllOnScreen('looksLikeThis.png'))
[(863, 117, 70, 13), (623, 137, 70, 13), (853, 577, 70, 13), (883, 617, 70, 13), (973, 657, 70, 13), (933, 877, 70, 13)]

locateCenterOnScreen（）函数只返回图像在屏幕上的中间位置的XY坐标：

pyautogui.locateCenterOnScreen('looksLikeThis.png')  # returns center x and y
(898, 423)

如果在屏幕上找不到图像，这些函数将返回None。

注意：定位功能很慢，可能需要整整一两秒钟的时间。
